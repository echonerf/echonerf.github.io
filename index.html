In submission to NeurIPS 2025

1. EchoNeRF: Generalizing NeRFs to multipath signals

A camera pixel (left) primarily captures light arriving directly from the object in front of it—along the line-of-sight (LoS) path. In contrast, an RF sensor (right) such as a WiFi receiver collects signals that include multiple reflections, or “echoes,” from walls and other objects in the surrounding environment. EchoNeRF is designed to learn from such multipath signals, enabling it to “see” the environment and infer spatial structure from sparse WiFi measurements.

2. What applications can EchoNeRF solve?

While EchoNeRF can be viewed as a general framework, as an application, we show how it can be used to solve a floorplan estimation problem in the above figure. (a) User's device measures wireless signal power. (b) Transmitted signals arrive at the device along a line of sight (LoS) path and after reflections from surrounding walls. (c) Our LoS model estimates crude walls. (d) Our first-order reflection model refines the inner and outer walls. (e) Together, the floorplan is estimated which can then be used to predict signals at new locations.

4. Can we improve EchoNeRF's output?

Surely! We can use several post-processing steps to improve the output of EchoNeRF. Below, we show the raw images from EchoNeRF and the post-processed images—through traditional image processing and via diffusion models.

Raw images from EchoNeRF

Post-processed images (image processing)

The estimated floorplan images are first binarized and then refined by applying morphological closing to fill small white gaps, followed by dilation to expand wall thickness. A median blur is used to smooth the edges, and finally contour detection is applied to remove small black regions based on area.

Post-processed images (diffusion models)

We can also use generative priors to improve the output of EchoNeRF. We first train a diffusion model (DDPM) on the ZInD dataset, which contains clean floorplans. Once trained, we perform denoising only over the final steps of the diffusion process, starting from the EchoNeRF's output inserted at time step \( t = 100 \). 5. What (other) forward tasks can EchoNeRF solve?

To demonstrate that EchoNeRF solves the true inverse problem, we perform the following tasks:

  • Channel Impulse Response (CIR) Prediction: We randomly select a Tx-Rx pair inside the floorplan and generate the CIRs using the NVIDIA's Sionna Simulator. This is done for both the ground truth and post-processed floorplans.
  • Basic Ray tracing: We simulate higher-order reflections (up to order 3) using the Sionna engine. Ray tracing visuals below show Tx-Rx locations and echo paths for both ground truth and predicted floorplans.
